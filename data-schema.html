ModelRadar 数据结构说明

为确保外部系统、AI Agent 及开发者能够更好地理解并复用本站的 JSON 数据源，特此提供以下字段定义及数据机制说明。

核心数据源 /data/models.json

这是驱动 ModelRadar 的最核心文件，包含所有当前追踪的 AI 模型的最新价格、上下文限制等信息。

全局属性

  • schemaVersion当前 JSON 的结构版本（如 "1.0.0"）。
  • generatedAt数据抓取或生成完成的具体时间（ISO 8601 格式）。
  • effectiveDate当前价格的生效日期（通常取 generatedAt 的年月日）。
  • currency全局价格货币单位，当前固定为 "USD"。
  • billingUnit全局计费单位基准，当前固定为 "per_1m_tokens"。

models 数组字段 (单个模型对象)

  • id模型唯一标识（URL 友好格式，如 "openai-gpt-4o"）。
  • name模型展示名称（如 "GPT-4o"）。
  • provider归属的 AI 厂商名称。
  • inputPriceUsdPer1M每百万 Tokens 的输入（Prompt）价格，单位为 USD。
  • outputPriceUsdPer1M每百万 Tokens 的输出（Completion）价格，单位为 USD。
  • cacheWritePriceUsdPer1M每百万 Tokens 的缓存写入价格（部分厂商支持）。
  • cacheReadPriceUsdPer1M每百万 Tokens 的缓存读取价格（部分厂商支持）。
  • contextWindow最大上下文长度（Token 数量，例如 128000）。
  • sourceUrl价格信息的官方来源 URL 页面，可用于数据校验。
  • updatedAt该条模型记录最近一次更新的日期时间（ISO 8601 格式）。
关于 null 值的说明：
如果 inputPriceUsdPer1M 或其他数值字段为 null，表示厂商官方未公开该字段的价格、暂时无法解析，或该模型不适用此计费维度。建议下游解析器做好防空处理。

JSON 结构示例

{
  "schemaVersion": 1,
  "generatedAt": "2026-05-17T00:00:00Z",
  "effectiveDate": "2026-05-17",
  "currency": "USD",
  "billingUnit": "per_1m_tokens",
  "models": [
    {
      "id": "deepseek-v4-flash",
      "name": "DeepSeek-V4-Flash",
      "provider": "DeepSeek",
      "inputPriceUsdPer1M": 0.14,
      "outputPriceUsdPer1M": 0.28,
      "cacheWritePriceUsdPer1M": null,
      "cacheReadPriceUsdPer1M": 0.014,
      "contextWindow": 128000,
      "sourceUrl": "https://www.deepseek.com/pricing",
      "updatedAt": "2026-05-17T00:00:00Z"
    }
  ]
}

变更日志 /data/changelog.json

记录每次价格更新的增量变动，主要用于渲染历史记录页面。

  • latest最近一次更新周期内发生的所有字段变动（包含涨价、降价、新模型等）。
  • history历史记录快照（按更新时间归档）。
  • summary统计摘要，如降价次数、新增模型数等。

历史快照 /data/history/YYYY-MM-DD.json

每次自动抓取更新完成后，系统都会将当次的 models.json 按 effectiveDate 作为文件名，额外保存一份静态快照（例如 2026-05-17.json）。这允许研究者和开发者回溯过去的具体价格状态。

站点地图 /sitemap.xml

站点地图是根据 /data/models.json 动态生成的。它不仅包含了所有的 HTML 页面（包括所有模型和对比的动态落地页），还将 models.json、changelog.json 及当日生成的历史快照直接列在 sitemap 中。所有条目的 lastmod 均与 effectiveDate 保持一致。

数据使用建议

您可以直接请求本站的 /data/models.json 并用于您自己的 AI 应用或成本监控中。但请注意：

  • 我们尽力确保价格的及时性，但在做出重大商业规划前，建议通过数据中的 sourceUrl 访问各厂商的官方价格页以进行最终核对。
  • 实际产生的计费和扣费标准应始终以您所在厂商的官方账单系统为准。
  • 建议同时读取并解析 schemaVersion 以便应对未来的字段演进。
