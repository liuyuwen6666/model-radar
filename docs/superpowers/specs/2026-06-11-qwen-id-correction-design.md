# 阿里通义（Qwen）模型 ID 数据订正设计规约

本文档说明如何将阿里通义 Qwen 3 错误的带点 ID 纠正为标准的连字符 ID，并彻底解决主库和历史快照中活跃（live）与遗留（legacy）模型记录共存的冗余问题。

## 变更背景
因历史版本的抓取解析规则原因，阿里百炼平台上的部分 `Qwen3-*` 模型在写入数据库时，其 ID 被误转换为了点号分隔的形式（如 `qwen3.32b`、`qwen3.8b`），甚至导致 `1.7B` 规模模型被转成了 `qwen3.1-7b`。
在后续规则升级后，最新的爬虫产生的是正确的连字符 ID（如 `qwen3-32b`）。这导致数据站中目前存在同一个模型的两份重复记录（一份 live 状态，一份 legacy 状态）。需要对历史库与主库做统一的数据迁移与去重合并。

## 设计目标
1. 编写迁移订正脚本 `scripts/migrate-qwen.js`，批量处理所有的 JSON 快照和主配置，将点号 ID 规整为连字符 ID。
2. 在迁移过程中对每个文件应用“活跃优先（live）”的 ID 去重合并机制，清除已下线归档的 legacy 重合项。
3. 保持通义千问 Scraper 逻辑及其他厂商数据不受影响。

## 订正 ID 映射表
* `qwen3.235b-a22b` ➔ `qwen3-235b-a22b`
* `qwen3.32b` ➔ `qwen3-32b`
* `qwen3.30b-a3b` ➔ `qwen3-30b-a3b`
* `qwen3.14b` ➔ `qwen3-14b`
* `qwen3.8b` ➔ `qwen3-8b`
* `qwen3.4b` ➔ `qwen3-4b`
* `qwen3.1-7b` ➔ `qwen3-1.7b`
* `qwen3.0-6b` ➔ `qwen3-0.6b`

## 修改范围
* **`data/models.json`**：替换 ID，并对 ID 去重（保留 `live` 记录）。
* **`data/sources.json`**：替换通义千问模型 ID 的配置。
* **`sitemap.xml`**：同步替换对应的静态页面链接。
* **`data/history/*.json`**：遍历 28 个历史 JSON 文件，对其中含有的旧 ID 进行迁移和去重覆写。

## 验证计划
1. 校验 `data/models.json` 等文件中是否还有残留的 `qwen3.32b`、`qwen3.8b` 字段。
2. 校验合并去重后模型数量和活跃状态是否正确。
3. 校验 git diff 确保没有网络测试的脏数据混入。
