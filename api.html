ModelRadar JSON API

ModelRadar 提供公开、免鉴权的静态 JSON API 数据源。开发者和 AI Agent 可以直接拉取我们的数据，将其整合到自动化脚本、应用或成本监控系统中。

可用端点 (Endpoints)

本站采用完全静态化部署架构。所有的 API 均为静态 JSON 文件，可通过标准的 HTTP GET 请求获取，无需任何 API Key 或鉴权认证。

GET /data/models.json
  • 用途：获取全量最新模型价格、上下文及官方来源信息。
  • 返回格式：包含全局计量单位（如 per_1m_tokens）和 models 数组的 JSON 对象。
  • 更新时间：每日自动抓取更新。
  • 适合场景：成本计算器数据源、最新模型价格查询。
GET /data/changelog.json
  • 用途：获取最新的价格变化记录摘要和历史变更清单。
  • 返回格式：包含 latest、summary 和 history 数组的 JSON。
  • 更新时间：随数据抓取同步生成。
  • 适合场景：追踪最新降价/涨价信息，渲染变更动态墙。
GET /data/history/YYYY-MM-DD.json
  • 用途：获取指定日期当天的 models.json 完整数据快照。
  • 返回格式：与 models.json 完全一致的 JSON 格式。
  • 更新时间：对应日期生成，固定不变。
  • 适合场景：历史成本对比、回测旧计费标准。
GET /sitemap.xml
  • 用途：获取全站页面地图（XML 格式），包含了所有的 JSON 数据源和历史快照的绝对 URL。
  • 返回格式：符合 sitemaps.org 标准的 XML。
  • 适合场景：搜索引擎抓取、自动化爬虫入口发现。

使用示例

JavaScript Fetch 示例

获取全量模型数据，筛选出某个厂商的模型，并找出输入价格最低的模型。

async function fetchCheapestModel() {
  const url = 'https://modelradar.cn/data/models.json';
  
  try {
    const response = await fetch(url);
    const data = await response.json();
    
    // 筛选 DeepSeek 的所有模型
    const deepseekModels = data.models.filter(m => m.provider === 'DeepSeek');
    
    // 找出输入价格最低的模型（排除 null）
    const validModels = deepseekModels.filter(m => m.inputPriceUsdPer1M !== null);
    validModels.sort((a, b) => a.inputPriceUsdPer1M - b.inputPriceUsdPer1M);
    
    if (validModels.length > 0) {
      const cheapest = validModels[0];
      console.log(`最便宜的模型：${cheapest.name}`);
      console.log(`输入价格：$${cheapest.inputPriceUsdPer1M} / 1M tokens`);
    }
  } catch (error) {
    console.error('获取数据失败:', error);
  }
}

fetchCheapestModel();

cURL 示例

通过命令行快速拉取并使用 jq 提取模型名称列表：

curl -s https://modelradar.cn/data/models.json | jq '.models[].name'

使用注意事项与最佳实践

  • 数据仅供参考：所有价格数据通过爬虫或自动化脚本从各厂商公开 pricing 页面采集。在做出关键业务或商业部署决策前，务必通过数据中的 sourceUrl 前往官方页面核对。
  • 计费单位标准：数据内的所有货币单位目前统一为 USD（美元），默认计费数量标准为 每 100 万 Tokens（per_1m_tokens）。
  • 合理缓存：由于 API 实际是静态文件，且更新频率通常为一天一次，强烈建议在您的系统中增加合理时间的本地缓存（例如 1 小时或更久），以提升拉取速度并降低不必要的网络开销。
  • 处理 Null 值：在解析时需留意部分计费字段（例如 cacheWritePriceUsdPer1M 或 contextWindow）可能是 null，请在代码逻辑中做好兜底处理。
